# Local Maestro

**Offline Portfolio Correlation & Risk Analysis** — a local recreation of [MyMaestro.co](https://mymaestro.co) for analyzing Composer trading strategies without needing public strategy IDs.

No strategy count limits. Uses your own API keys to pull any strategy — public or private.

## Accuracy

Validated against MyMaestro.co across 5 strategies of varying complexity (5–15+ IF nodes). Results on the same date range:

| Metric | Accuracy vs MyMaestro |
|--------|----------------------|
| Sharpe Ratio | ~98% (within 0.02-0.06) |
| Sortino Ratio | ~98% (within 0.02-0.12) |
| Volatility | ~99.9% |
| Max Drawdown | ~99% |
| Correlation Matrix | ~99.9% |
| Cumulative Return | ~95-99% (cache timing) |
| Calmar Ratio | ~97% |

Remaining small gaps in cumulative return are from backtest cache timing — MyMaestro fetches live from Composer's API on each request.

## Features

All 5 analysis tabs from the original Maestro:

| Tab | Contents |
|-----|----------|
| **Returns** | Cumulative returns (linear/log), rolling returns, rolling win rate, underwater (drawdown) plot |
| **Correlations** | Correlation matrix heatmap, drawdown correlation matrix, rolling mean correlation, rolling CARP, rolling Smart CARP |
| **Volatility** | Rolling volatility, rolling Sharpe ratio, rolling Sortino ratio |
| **Exposure** | Asset holding metrics, allocation history (when Composer data available) |
| **Metrics** | Summary table with all metrics per strategy + portfolio |

**Key metrics:**
- **CARP** (Correlation And Risk-adjusted Performance) = Sortino / (1 + mean_correlation)
- **Smart CARP** = Sortino / (1 + mean_drawdown_correlation)

**Advantages over MyMaestro.co:**
- No 20-strategy limit
- Works with private (non-public) strategies
- Offline analysis — no data leaves your machine
- Multiple input formats (CSV, Composer JSON, Rainboy HTML, strategy JSON auto-backtest)
- Interactive web server mode with date range picker

## Quick Start

```bash
# Install dependencies (one-time)
pip install pandas numpy plotly

# Analyze a CSV of strategy equity curves
python maestro.py sample_data/sample_4_strategies.csv

# Analyze Composer backtest cache files
python maestro.py backtest1.json backtest2.json backtest3.json

# Analyze all files in a directory
python maestro.py --dir ./my_backtests/

# Custom date range and weights
python maestro.py file1.json file2.json --start 2023-01-01 --weights 0.6 0.4
```

The report opens automatically in your browser.

## Setup

### Environment Variables

For fetching backtests from Composer's API (optional — you can also use pre-downloaded CSV/JSON files):

```bash
export COMPOSER_API_KEY="your-api-key-id"
export COMPOSER_API_SECRET="your-api-secret"
```

For the interactive web server (`server.py`):
```bash
export COMPOSER_ACCOUNT_UUID="your-account-uuid"
```

Alternatively, create a `.env` file in your project root with these values.

## Input Formats

### 1. CSV File (simplest)
A CSV with date column + one equity column per strategy:
```csv
date,Strategy A,Strategy B,Strategy C
2023-01-03,10000,10000,10000
2023-01-04,10050,9980,10020
...
```

### 2. Composer Backtest Cache JSON
The `dvm_capital` format from Composer's backtest API. These can be fetched using the included helper script:
```bash
# Fetch backtests by symphony ID
python fetch_backtests.py ENtpwkO1bLCFmhSMjjgp ygUqoICgwIZTaOyCzmAG

# Fetch and immediately analyze
python fetch_backtests.py ID1 ID2 ID3 --analyze
```

### 3. Rainboy Backtester HTML Reports
Parse HTML reports generated by the Rainboy CLI Backtester.

### 4. Strategy JSON Files (auto-backtest)
Provide Composer strategy JSON files and let Rainboy backtest them:
```bash
python maestro.py --backtest strategy1.json strategy2.json
```
Requires the Rainboy CLI Backtester to be installed.

## Interactive Web Server

```bash
python server.py --port 8080
```

Provides a browser UI with:
- Strategy selection from your cached backtests
- Date range picker (All Time, 3Y, 1Y, YTD, 6M, 3M, Custom)
- Live analysis dashboard

## Options

| Flag | Description |
|------|-------------|
| `--dir`, `-d` | Directory of input files |
| `--start`, `-s` | Start date (YYYY-MM-DD) |
| `--end`, `-e` | End date (YYYY-MM-DD) |
| `--weights`, `-w` | Portfolio weights (decimal, must sum to 1.0) |
| `--window` | Rolling window size in trading days (default: 21) |
| `--output`, `-o` | Output HTML file path |
| `--no-open` | Don't open report in browser |
| `--backtest`, `-b` | Auto-backtest strategy JSONs via Rainboy |

## Architecture

```
maestro.py              # CLI entry point
server.py               # Interactive web server
lib/
  data_loader.py        # Load data from CSV, Composer JSON, Rainboy HTML
  analytics.py          # Core computation engine (pandas/numpy)
  report.py             # HTML report generator (Plotly.js)
fetch_backtests.py      # Helper to fetch Composer backtests
generate_sample_data.py # Generate sample data for testing
sample_data/            # Sample input files
backtest_data/          # Cached backtests (gitignored)
reports/                # Generated reports (gitignored)
```

## How It Works

1. **Load** — Reads backtest data from various input formats, extracting daily equity curves
2. **Align** — Finds the overlapping date range across all strategies, rebases equity to common start. When a start date is specified, includes the previous trading day so the return on the start date is captured (matches MyMaestro.co).
3. **Analyze** — Computes daily returns, portfolio simulation (daily rebalancing to target weights), correlations, rolling statistics
4. **Report** — Generates a self-contained HTML file with interactive Plotly.js charts
